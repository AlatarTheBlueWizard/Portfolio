Local Storage Snippets and Explanation

The displayArea function first identifies the value of the text inputted by the user in the textarea.
That value is then placed within the section (story_editor) and displayed to the user.


The loadStory function first identifies the value of the text inputted by the user in the input field
(in this case the story title). Thanks to the localStorage.setItem property from saveStory applying
a key-value pair, the getItem property for localStorage in loadStory is able to find the saved/applicable
title (if it matches) and then displays it's story editor value in the story editor textarea using innerHTML.


The saveStory function first identifies the value of the text inputted by the user in the input field.
The function then identifies the text value in the story editor textarea. The name and value variables
are then saved into a key-value pair using the setItem property of localStorage (which is then utilized in loadStory).


The section tags are used in order to force the buttons to be on a seperate line from the input field by taking the whole
width of the page, hence making the output more readable. The button onclick attributes call the js functions, hence
giving them the ability to perform the desired functionality once clicked.
